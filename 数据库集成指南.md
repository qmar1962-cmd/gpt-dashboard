# GPT 每日通报可视化看板 - 数据库集成指南

## 🎉 系统升级完成！

系统已升级为**完整的数据库驱动应用**，支持 6 种数据模板的上传、存储、查询和趋势分析。

---

## 📊 支持的 6 种数据模板

### 1️⃣ 岗位效能异常
**文件示例：** `岗位效能异常_2026-04-24.csv`

**必需字段：**
- 数据日期
- 省区
- 中心
- 岗位名称（卸车/装车/倒包/供件/封包/分拣/扫描）
- 目标值
- 当月人均日绩效
- 目标偏离（%）

**可选字段：**
- 全网同岗均值
- 均值偏离（%）

---

## 🚀 使用流程

### 步骤 1：打开数据管理中心
1. 点击左侧边栏的 **蓝色上传按钮**（带红色提示点）
2. 进入数据管理界面

### 步骤 2：选择数据模板
1. 在 6 种模板中选择对应类型
2. 系统会显示模板说明和必需字段

### 步骤 3：上传文件
1. 拖拽 Excel/CSV 文件到上传区域
2. 或点击选择文件
3. 系统自动解析并验证数据

### 步骤 4：数据存储
1. 数据自动保存到本地数据库（localStorage）
2. 按日期 + 中心 + 数据类型组织
3. 自动保留 30 天数据

### 步骤 5：查看数据
1. 上传完成后自动切换到仪表盘
2. 数据实时更新到图表和表格
3. 点击数据模块可查看近 7 天趋势（待实现）

## 💾 数据存储说明

### 存储位置
- **当前使用：** localStorage（浏览器本地存储）
- **容量限制：** 约 5MB
- **持久性：** 数据保存在浏览器，清除缓存会丢失

### 数据结构
```
每日数据 (daily_data)
├─ 2026-04-24
│  ├─ centers
│  │  ├─ hubei-wuhan
│  │  │  ├─ job_performance (岗位效能)
│  │  │  ├─ salary_performance (薪资绩效)
│  │  │  ├─ attendance_15days (连续 15 日出勤)
│  │  │  ├─ attendance_7days (七日未出勤)
│  │  │  ├─ center_scope (中心管幅)
│  │  │  └─ center_headcount (在职人数)
│  │  └─ hunan-changsha
│  │     └─ ...
│  └─ uploadTime: 时间戳
└─ 2026-04-23
   └─ ...

中心数据 (center_data) - 用于趋势查询
├─ hubei-wuhan_job_performance_2026-04-24
├─ hubei-wuhan_job_performance_2026-04-23
└─ ...
```

### 数据清理
- **自动清理：** 超过 30 天的数据自动删除
- **手动清理：** 点击"清理过期数据"按钮
- **清理范围：** 删除 30 天前的所有数据

## ⚠️ 注意事项

1. **日期格式** - 使用 `YYYY-MM-DD` 格式（如：2026-04-24）
2. **省区名称** - 使用标准名称（如：湖北省区、湖南省区）
3. **中心名称** - 使用标准名称（如：武汉、长沙）
4. **数值格式** - 不要包含特殊字符（如：%、¥）
5. **文件大小** - 建议不超过 1MB
6. **编码格式** - CSV 使用 UTF-8 编码
